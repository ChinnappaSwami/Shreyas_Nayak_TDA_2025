Trees are a connected acyclic graph. [connected meaning all pair of nodes are "reachable" or have a path][Acyclic meaning there are no cycles present i.e. the shape is not 'enclosed' in any subset of the graph ]

Trees have a root node [ the topmost node ]. Trees have a Parent node [ nodes which have children i.e. nodes which is always present in the path from the root node to the child nodes ]. Trees have leaf nodes [ leaf nodes meaning nodes who doesn't have any children i.e. only 1 edge connected to leaf node's parent is present ]

This data structure is quite useful in managing a LOT of things and i mean a LOT. This is how your Databases work, This is how your Ordered maps work, this is how Sets work, This helps in faster sorting, This helps in faster Ranged Queries ,This helps in faster string comparision, this is also used in some search engines ,this is also used in Descision Trees & Random Forests & XGBoosts in AIML, and a lot of other stuff!

3 Main ways to store this tree is used
1) Treat it as a normal graph, and use adjacency matrix or list.
2) Create a structure like linked list where each node stores:- value and a linked list or vector of pointers to child nodes.
3) store it in a array. [ will explain later how to. ]
