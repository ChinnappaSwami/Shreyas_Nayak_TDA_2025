Recursions are basically a function calling itself some number of times. But why?? So a simple rule that we have seen from so many years is divide and rule! similarly all we do in this is "make the question smaller", Think like a chess board, if you want to find the best move in a position, you can only do it if the next sequence of moves leads to a win! Think of a function that we need to solve for some value n, for doing that lets understand the question with smaller numbers, "solve the function with value n-1", then "solve it using n-2", and so on and finally "solve it using value 1", if we know how to solve it for smaller numbers, then we can slowly increase it to a larger scale!!
This is the general idea of recursion, when you were kids, you studied multiplying small numbers less than 10 like 2*3, 4*5, etc but later we studied multiplication with larger numbers! this is the idea, we train our computers with smaller values, then go with larger ones! Now how much larger depends on different methods & approaches!
